### Categories

We use these categories for changes:
- **Added** - New features, documentation, examples
- **Changed** - Changes to existing functionality
- **Deprecated** - Features that will be removed
- **Removed** - Removed features
- **Fixed** - Bug fixes
- **Security** - Security vulnerability fixes

## Release Process

### Version Numbers

We follow semantic versioning:
- **Major** (1.0.0): Breaking changes, major restructuring
- **Minor** (0.1.0): New features, content additions
- **Patch** (0.0.1): Bug fixes, minor improvements

### Release Checklist

Before releasing a new version:

- [ ] All planned features complete
- [ ] Documentation reviewed
- [ ] Examples tested
- [ ] Links validated
- [ ] Changelog updated
- [ ] Version numbers updated
- [ ] README reflects current state
